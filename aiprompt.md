# AI 투두 생성 규칙 (aiprompt.md)

> 이 문서는 slowgoes의 **AI 투두 자동생성** 품질 규칙의 단일 기준이다.
> 구현: `src/lib/ai/analyze.ts`의 `generateTodoSuggestions` 프롬프트가 이 규칙을 따른다.
> **규칙을 바꾸면 이 문서와 프롬프트를 함께 수정한다.**

## 목적

유저가 "지금 바로 시작할 수 있는" 질 좋은 투두 3개를 제안한다.
완벽한 계획이 아니라 **70점짜리 행동** — 어설퍼도 오늘 착수할 수 있는 행동 — 을 지향한다.
(앱의 목적: 하나의 버킷에 집중해 행동력·실천력을 높이는 것 — CLAUDE.md Philosophy)

## 컨텍스트 소스 (프롬프트에 주입하는 정보)

| 소스 | 역할 | 없을 때 |
|---|---|---|
| ① **버킷 타이틀 + 삶의 영역** | 무엇을 위한 행동인지 (필수) | — |
| ② **지향점 3단** (언젠가 / 올해안 / 해당 달) | 방향의 근거. **투두는 "해당 달" 발걸음에 직결**되고, 언젠가·올해안은 톤을 잡는 배경 | 있는 것만 사용 |
| ③ **MBTI** (personality_type) | 행동 스타일 조정 — I형: 혼자 시작 가능한 행동 우선 / E형: 사람과 연결되는 행동 허용 / J형: 계획·정리형 / P형: 즉흥·탐색형 | 생략 |
| ④ **나이** (life_clock_age) | 인생 단계 맥락 (표현·난이도 조정) | 생략 |
| ⑤ **최근 일기 3건 발췌** (plain_text 앞 200자) | 현재 관심사·감정 상태 반영 — 일기에 드러난 상황이 있으면 1개는 그것과 연결 | 생략 |
| ⑥ **기존 투두 제목 전체** | **중복 금지** 목록 | 필수(빈 목록 가능) |
| ⑦ **기준 날짜** (캘린더 선택 날짜) | "그 날" 실행 가능한 행동인지 판단 | 오늘 |

## 품질 규칙 (7가지)

1. **구체적 행동 동사로 시작** — "~에 대해 알아보기"(추상) ✕ → "~ 검색해서 후보 3개 적기"(구체) ○
2. **30~60분 실행 단위** — 하루 안에, 한 번에 끝낼 수 있는 크기
3. **70점 지향** — 완벽한 준비·조사보다 어설픈 착수를 우선하는 문장
4. **완료 판정 가능** — 개수·시간·산출물이 문장에 있어 체크 가능할 것 (예: "3곳", "10분", "1개 작성")
5. **버킷·지향점 직결** — 해당 달 발걸음을 실제로 전진시키는 행동만
6. **일기 반영(있으면)** — 최근 일기의 관심사/막힘과 연결되는 항목 1개 포함
7. **접근 다양화** — 3개가 서로 다른 유형이 되게: (조사·준비형) / (실행·산출형) / (사람·환경 연결형) 중 조합

## 출력 형식

- **정확히 3개**, 한국어, 각 40자 이내 권장
- JSON만 응답:

```json
{ "todos": [ { "title": "..." }, { "title": "..." }, { "title": "..." } ] }
```

## 금지

- 기존 투두 제목과 의미가 겹치는 항목
- "매일 ~하기" 같은 반복 전제 문장 (반복 설정은 유저가 [반복] 버튼으로 결정)
- 동기부여 문구·이모지·설명 — 행동 문장만
